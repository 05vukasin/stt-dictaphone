# PWA

Installable, offline-capable, served from the same Next.js app.

## Manifest

`public/manifest.json` declares:

- `name`: "STT Dictaphone"
- `display`: `standalone`
- `start_url`/`scope`: `/`
- `theme_color`/`background_color`: `#161616` (matches dark canvas)
- Icons: 192, 512, and 512 maskable

Linked from the document via `metadata.manifest = "/manifest.json"` in
`src/app/layout.tsx`.

## Icons

Generated by `scripts/generate-icons.mjs` (run via `npm run icons`). It draws
a microphone glyph in dirty-white on dark grey via inline SVG, then rasterizes
to PNG with `sharp`. Sizes:

- `public/icons/icon-192.png` (any)
- `public/icons/icon-512.png` (any)
- `public/icons/icon-maskable-512.png` (maskable; extra padding so the mark
  survives platform safe-area cropping)
- `public/icons/apple-touch-icon.png` (180×180 for iOS)

## Service worker

We use `@serwist/next` (modern, App Router-native; the spiritual successor to
next-pwa).

- Source: `src/app/sw.ts`. It instantiates a `Serwist` with
  `precacheEntries: self.__SW_MANIFEST` (injected at build), `skipWaiting`,
  `clientsClaim`, navigation preload, and the default runtime cache.
- Output: `public/sw.js` written by the webpack plugin during `next build`.
  Disabled in dev (`disable: process.env.NODE_ENV === "development"`).

### Build pipeline note

`next build` defaults to **Turbopack** in Next 16. @serwist/next is webpack-
based and can't hook into Turbopack today. Both `npm run dev` and
`npm run build` pass `--webpack` to force the webpack pipeline. Without that
flag, the SW does not get generated and the app silently degrades to non-PWA.

## Registration

`<ServiceWorkerBoot>` in `src/app/providers.tsx` registers `/sw.js` on mount
(production only). Failures are swallowed — a missing SW must never break the
app.

## Install prompt

`src/components/pwa/install-prompt.tsx` listens for `beforeinstallprompt`
(Chrome/Edge), stashes the event, and renders a custom toast-style banner
with an "Install" button. Once the user dismisses it, we set
`stt-dict:install-dismissed=1` in localStorage and never show it again.

iOS Safari doesn't fire `beforeinstallprompt`, so iOS users add the app via
Share → "Add to Home Screen". The `apple-mobile-web-app-capable` and
`apple-touch-icon` metadata in `layout.tsx` make this work cleanly.

## Offline behavior

- The shell (HTML, JS, CSS, icons) is precached at install — opens offline.
- Recording works offline (it's all client-side).
- IndexedDB persists across offline sessions.
- Transcription/summary calls require network. If they fail, the toast shows
  the error and the transcript stays in `error` status — the user can re-run
  from the recording detail page once they're back online. (A queued retry
  worker is on the [ROADMAP](./ROADMAP.md).)

## Verifying

After `npm run build && npm start`:

1. DevTools → Application → Manifest: should detect "STT Dictaphone".
2. DevTools → Application → Service Workers: should show `/sw.js` activated.
3. DevTools → Application → Storage → IndexedDB: should show
   `stt-dictaphone` after the first recording.
4. Lighthouse → PWA: target ≥90.

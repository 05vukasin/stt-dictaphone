# Testing

`npm test` runs the Vitest suite once. `npm run test:watch` is interactive.

## Setup

`vitest.config.ts`:

- `environment: "jsdom"` — DOM globals + a Window.
- `globals: true` — `describe`/`it`/`expect` are global.
- Path alias `@/` → `./src/` (matches tsconfig).
- `setupFiles: ["./vitest.setup.ts"]` injects:
  - `@testing-library/jest-dom/vitest` — `toBeInTheDocument`, etc.
  - `fake-indexeddb/auto` — IDB polyfill for jsdom.
  - `cleanup()` after each test.
  - Stubs for `matchMedia` and `ResizeObserver` (jsdom omits them).

## What's deliberately not covered

- **The full WaveCanvas component** — the React wrapper depends on
  `requestAnimationFrame`, `ResizeObserver`, `AudioContext`, and a real canvas
  context. The PURE renderer functions are tested. A snapshot-of-the-React-tree
  test would test the render scaffolding (which is trivial) and miss the actual
  visual behavior.
- **Recording end-to-end** — getting from "click button" through getUserMedia,
  AudioContext, MediaRecorder, IDB, and an HTTP call is integration territory
  best done with Playwright in a real browser. v1 ships unit tests only.
- **Service worker behavior** — generated by serwist; trusting their integration
  tests rather than re-running them.

## Pitfalls and gotchas

- **jsdom's `Request.formData()`** is incomplete — the `transcribe` route test
  stubs the formData method on the Request object directly to bypass it.
- **fake-indexeddb deadlocks on `deleteDatabase`** if a connection is still open.
  The `__resetDbForTests()` helper closes the cached connection first, then the
  test runs `indexedDB.deleteDatabase(...)` to wipe state.
- **next-themes** is mocked in the theme-selector test to keep the test focused
  on UI behavior rather than the next-themes implementation.
- **`MediaRecorder` mock** — `recorder.test.ts` provides a class with the
  shapes the recorder uses (`ondataavailable`, `onstop`, `start/pause/resume/stop`).
  When upgrading vitest or adding new recorder methods, extend the mock too.

## Coverage

Configured in `vitest.config.ts`:

- Provider: `v8`
- Includes: `src/lib/**`, `src/components/**`
- Excludes: `**/*.test.*`, `**/*.d.ts`

Run `npx vitest run --coverage` to generate a report. Target ≥85% on `lib/`.
